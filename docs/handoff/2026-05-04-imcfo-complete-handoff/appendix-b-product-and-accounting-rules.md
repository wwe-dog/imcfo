# 附录 B：产品与会计规则

## 1. 最高产品规则

- 我为 CFO 是普通自然人的个人经营 App，不是普通流水账。
- V0.1 只做普通自然人，不做个体工商户、超级个体、企业主体或家庭合并账本。
- 用户可见文案必须中文。
- MVP 不引入后端、登录、数据库、云同步、支付、AI/API、税务申报、VAT 或发票逻辑。

## 2. 六大会计要素

- 资产
- 负债
- 所有者权益
- 收入
- 费用
- 利润

核心公式：

```text
所有者权益 = 资产 - 负债
利润 = 收入 - 费用
资产 = 负债 + 所有者权益
```

## 3. 三大报表

- 资产负债表：总资产、总负债、所有者权益、资产负债率。
- 利润表：收入、费用、利润、储蓄率。
- 现金流量表：经营活动、投资活动、筹资活动、现金净变化。

简易版面向生活语言，专业版保留财务语言。两者共享同一套数据和计算逻辑。

## 4. 典型交易规则

- 工资到账：资产增加、收入增加、经营活动现金流入。
- 日常支出：费用增加、资产减少、经营活动现金流出。
- 信用卡消费：费用增加、负债增加、不产生现金流。
- 信用卡还款：资产减少、负债减少、筹资活动现金流出、不影响利润。
- 投资买入：投资资产增加、现金减少、投资活动现金流出、不直接影响利润。
- 投资卖出：投资资产减少、现金增加、投资活动现金流入，V0.1 暂不自动计算投资收益。
- 转账：账户间移动，不影响收入、费用、利润或现金净变化。
- 应收/应付确认：非现金事项。

## 5. 当前实现边界

- 基础公式集中在 `mobile/src/domain/accounting/calculations.ts`。
- 交易变更规则集中在 `mobile/src/domain/accounting/transactionRules.ts`。
- 对账规则集中在 `mobile/src/domain/accounting/reconciliationRules.ts`。
- 现金流默认分类在 `mobile/src/domain/accounting/cashFlowRules.ts`。
- 报表期间过滤在 `mobile/src/domain/accounting/periodFilters.ts`。

## 6. 后续必须补齐的验证样例

- 空数据：金额为 0，资产负债率和储蓄率不可计算。
- 期间边界：起止日期当天交易包含，跨月交易排除。
- 信用卡消费和还款：利润、负债、现金流分别验证。
- 转账：不改变利润和现金净变化。
- 投资买卖：投资现金流方向正确，不自动确认收益。
- 应收/应付：确认和收回/支付不能重复确认收入或费用。
- 退款、冲减、负数金额：需要先明确产品口径再测试。
