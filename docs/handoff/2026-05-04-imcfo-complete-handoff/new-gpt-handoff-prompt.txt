Product:
我为 CFO is a personal finance mobile MVP that treats an ordinary natural person as a company.
It is not a normal bookkeeping app.
The product should help the user understand personal finance through balance sheet, income statement, cash flow statement, operating conclusions, and net worth.

MVP rules:
- Use Expo + React Native + TypeScript + AsyncStorage.
- Do not add backend, login, cloud sync, database, payment, membership, AI/API integration, tax filing, VAT, invoice, or individual-business logic.
- Keep bottom navigation as 首页 / 管理 / 报表 / 我的 unless the user explicitly changes product scope.
- Keep all user-facing copy in Chinese.
- Keep accounting logic out of UI.
- Keep report calculation functions pure and testable.
- Do not let frontend code invent accounting formulas.
- Do not change storage schema, route keys, storage keys, parser behavior, transaction mapping, domain model fields, or accounting formulas unless explicitly requested.

Current visual system:
- The official visual direction is now IMCFO 暗黑液态 CFO 风格 / Dark Liquid CFO Style.
- One-line definition: 深色金融操作台 + 液态玻璃卡片球体 + 赛博 HUD 数据流 + 个人 CFO AI 入口.
- Orange remains a brand anchor, key action color, and warning color; it is no longer the only primary visual color.
- Financial semantic colors remain stable: positive/income/improvement/profit = green; negative/expense/risk/loss = red; warning/pending/focus = orange or amber; neutral amount/text = black, dark gray, or light gray; links/drilldowns/clickable = blue or brand accent.
- Home, sphere, smart record input, AI entry, and HUD may use dark liquid visual language.
- Management, accounts, assets/liabilities, transaction records, reports, and settings must preserve readability and accounting clarity.
- Read docs/standards/imcfo-visual-system.md before designing or implementing UI.
