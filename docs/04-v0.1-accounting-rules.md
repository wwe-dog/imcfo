# 我为 CFO · V0.1 会计规则

本文档定义 V0.1 的会计底座和个人场景适配规则。它不是企业法定财务报表规则，也不是正式税务或审计规则。

## 1. 核心原则

- V0.1 只处理普通自然人。
- 个人会计体系优先，企业会计准则作为重要参考来源。
- 会计逻辑必须独立于 UI。
- 专业版预留复式记账和借贷记账法思想。
- 简易版输入是产品交互，专业版分录是会计底座。

## 2. 六大会计要素

V0.1 使用：

1. 资产
2. 负债
3. 所有者权益
4. 收入
5. 费用
6. 利润

个人语境：

- 所有者权益 = 真正属于我的钱 = 资产 - 负债
- 利润 = 本期经营结果 = 收入 - 费用

## 3. 基础等式

必须保持：

- 资产 = 负债 + 所有者权益
- 利润 = 收入 - 费用

## 4. 记账基础

V0.1 采用现金流优先，准权责发生制预留。

含义：

- 用户优先记录“钱是否收到、花出、借入、还出”。
- 系统底层为应收、应付、预收、预付、折旧、摊销、减值预留结构。
- V0.1 不强制用户理解权责发生制。
- V0.1 不做复杂应计、递延、摊销、折旧和减值。

## 5. 复式记账预留

每笔经济事项应至少影响两个会计要素、账户或报表项目。

V0.1 必须预留：

- `JournalEntry`
- `JournalEntry.lines`
- `debit`
- `credit`
- `reportElement`
- `validateJournalEntryBalance`

原则：

- 有借必有贷，借贷必相等。
- UI 不允许临时发明会计处理。
- 异常交易必须进入 `06-v0.1-transaction-mapping.md`。

## 6. 资产确认

可以确认为个人资产的项目：

1. 用户可以支配。
2. 具有现金价值或使用价值。
3. 未来可能带来经济利益或降低未来支出。
4. 金额可以合理估计。

V0.1 不要求审计式确认。

## 7. 负债确认

可以确认为个人负债的项目：

1. 用户未来需要偿还。
2. 偿还会导致现金或资产流出。
3. 金额可以确定或合理估计。

信用卡必须作为负债账户处理。

## 8. 收入确认

V0.1 以用户实际收到或可明确记录的收入为主。

常见收入：

- 工资薪金
- 生活费
- 奖学金
- 兼职收入
- 劳务收入
- 投资收益
- 红包礼金
- 退款
- 其他收入

简单副业收入只做普通自然人收入记录，不进入经营所得核算。

## 9. 费用确认

V0.1 将个人支出中消耗当期资源的项目确认为费用。

常见费用：

- 生存费用
- 居住费用
- 餐饮费用
- 交通通讯费用
- 学习成长费用
- 娱乐消费
- 社交费用
- 健康医疗费用
- 财务费用
- 税费
- 其他费用

成长投入在会计上可以先作为费用处理，但在产品展示上应单独强调。

## 10. 投资与固定资产

买入股票、基金、理财不直接影响利润表。

它通常只是资产结构变化：

现金资产减少，投资资产增加。

只有以下情况影响利润表：

- 分红
- 利息
- 卖出产生收益或损失
- 后续版本启用估值变动规则

V0.1 固定资产可手动录入，不强制折旧。

## 11. 信用卡规则

信用卡消费：

- 费用增加
- 负债增加
- 不产生现金流出

信用卡还款：

- 负债减少
- 现金减少
- 产生筹资活动现金流出，或在个人适配口径中展示为债务偿还现金流出

这条规则必须进入报表计算和交易映射，不允许 UI 自行处理。

## 12. 税务边界

V0.1 可预留税务字段，但不做正式税务功能。

不得做：

- 正式个人所得税申报
- 税务筹划
- 节税建议
- 个体工商户经营所得核算
- 增值税计算
- 发票抵扣
- 纳税申报表生成
