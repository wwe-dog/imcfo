# 我为 CFO · V0.1 项目宪法

本文档是“我为 CFO”V0.1 的最高项目规则。后续产品设计、数据模型、报表计算、页面开发、测试和 Codex 任务拆解，都必须优先遵守本文档。

## 1. 产品定位

我为 CFO 不是普通记账 App。

我为 CFO 是一款把普通自然人的个人生活翻译成公司式财务报表的个人经营 App。它把用户视为一个“个人公司”，通过记录资产、负债、收入、费用和现金流，生成个人版资产负债表、利润表、现金流量表，让用户看清自己的财务全貌。

V0.1 的一句话定位：

像经营公司一样经营自己。

## 2. V0.1 最高边界

V0.1 只做：

普通自然人个人经营账本。

V0.1 不做：

- 个体工商户
- 超级个体模式
- 个人独资企业
- 合伙企业自然人合伙人
- 企业主体
- 家庭合并账本
- 经营所得核算
- 增值税管理
- 发票管理
- 正式税务申报
- 后端、登录、数据库、云同步
- AI 分析、投资建议、贷款建议、支付或会员

普通自然人、超级个体、个体工商户、家庭主体必须分层建模。V0.1 不允许把它们混成一套逻辑。

## 3. 会计主体

V0.1 的会计主体是一个普通自然人。

这个自然人把自己的个人生活、收入、支出、资产、负债和现金流，当作一家“个人公司”来管理。

V0.1 不把用户视为企业主、个体工商户、公司股东主体、工作室经营主体或家庭共同体。

## 4. 会计底座

V0.1 采用个人经营场景下的会计体系，参考中国企业会计准则体系和《中华人民共和国会计法》的基础思想，但不机械照搬企业会计准则。

原因：

企业会计准则服务企业主体、对外披露、监管、审计和投资者决策；我为 CFO 服务普通自然人的个人财务理解、经营复盘和现金流管理。

V0.1 采用六个基础会计要素：

1. 资产
2. 负债
3. 所有者权益
4. 收入
5. 费用
6. 利润

必须遵守：

- 资产 = 负债 + 所有者权益
- 所有者权益 = 资产 - 负债
- 利润 = 收入 - 费用

## 5. 三大报表

V0.1 必须支持三张报表：

1. 资产负债表：回答“我现在拥有多少，欠了多少，真正属于我的钱是多少？”
2. 利润表：回答“我这个期间赚了还是亏了？”
3. 现金流量表：回答“我的现金是怎么流进和流出的？”

现金流量表分为：

- 经营活动现金流
- 投资活动现金流
- 筹资活动现金流
- `nonCash`，用于不产生现金流的事项

## 6. 简易版与专业版

简易版和专业版不是两套系统。

必须坚持：

1. 底层数据只有一套。
2. 计算逻辑只有一套。
3. 展示语言分成两套。

简易版使用生活语言，例如：

- 真正属于我的钱
- 本月赚亏
- 钱去哪了
- 债务压力
- 成长投入

专业版使用财务语言，例如：

- 资产
- 负债
- 所有者权益
- 收入
- 费用
- 利润
- 经营活动现金流
- 投资活动现金流
- 筹资活动现金流

简易版可以口语化，但不能改变底层专业含义。

## 7. 复式记账预留

V0.1 用户界面不要求用户理解借贷记账法，也不要求用户手动录入会计分录。

但底层必须预留复式记账能力：

- `JournalEntry`
- `JournalEntry.lines`
- `debit`
- `credit`
- `reportElement`
- `validateJournalEntryBalance`

原则：

- 有借必有贷，借贷必相等。
- 同一笔交易在底层必须能映射为平衡的会计影响。
- UI 不能临时发明会计处理。
- 异常交易必须先进入交易映射规则，再进入实现。

## 8. 移动端存储边界

V0.1 最终产品形态是移动 App，使用 Expo + React Native + TypeScript + AsyncStorage。

V0.1 不做后端、不做登录、不做数据库、不做云同步。

移动端必须通过 storage adapter 访问本地存储。AsyncStorage 只负责本地存取，不参与报表计算。

报表计算函数必须：

- 输入数据，输出结果。
- 不直接读取 AsyncStorage。
- 不直接写入 AsyncStorage。
- 不依赖 React 状态。
- 可单独测试。

## 9. 页面边界

V0.1 只做 5 个页面：

1. 首页 Dashboard
2. 记一笔 Record
3. 资产负债 Assets & Liabilities
4. 报表 Reports
5. 设置 Settings

页面结构详见 `docs/02-v0.1-page-structure.md`。

## 11. 税务边界

V0.1 可以预留税务字段，但不做正式税务功能。

V0.1 可记录税前税后收入、个税、社保、公积金等信息，但不得：

- 生成纳税申报表
- 给出税务筹划建议
- 给出节税或避税建议
- 处理个体工商户经营所得
- 计算增值税
- 管理发票抵扣
- 输出税务机关可提交文件

涉及正式申报、税务争议、复杂经营所得、境外所得、个体工商户、个人独资企业等场景时，用户应以税务机关官方口径和专业意见为准。

## 12. 数据安全与隐私

V0.1 默认数据只存储在用户本地浏览器。

V0.1 不收集：

- 身份证号
- 银行卡号
- 家庭住址
- 人脸信息
- 通讯录
- 精确定位
- 真实征信信息

用户必须可以导出数据和清空本地数据。清空数据必须二次确认。

## 13. 开发原则

1. 先跑通闭环，再追求完整。
2. V0.1 只做普通自然人。
3. 会计逻辑和 UI 必须分离。
4. 报表计算必须是纯函数。
5. 用户看到中文，技术命名可以使用 English。
6. 不为了炫技增加复杂度。
7. 不把产品做成会计考试软件。
8. 不把产品做成普通流水账。
9. 每个功能都必须服务“个人经营”主线。
10. 太复杂的能力进入后续版本，不进入 V0.1。

## 14. V0.1 最终确认

V0.1 的核心是：

- 一个主体：普通自然人
- 六个要素：资产、负债、所有者权益、收入、费用、利润
- 三张报表：资产负债表、利润表、现金流量表
- 两种视角：简易版、专业版
- 一个存储：AsyncStorage，经由 storage adapter 访问
- 一个闭环：记录数据，生成报表，看清个人财务全貌

偏离这个闭环的功能，暂时不做。
