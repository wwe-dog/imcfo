# 我为 CFO · V0.1 数据模型

本文档定义 V0.1 的 TypeScript 数据对象和移动端存储边界。实现时以简单、可计算、可迁移为优先。

## 1. 通用规则

- 所有持久化对象必须有 `id`。
- 所有持久化对象必须有 `createdAt` 和 `updatedAt`。
- 金额默认使用人民币。
- AsyncStorage 只能通过 storage adapter 访问，不参与报表计算。
- 报表计算函数只接收显式输入，不直接读取 AsyncStorage。
- 导出数据必须包含 `version`。

## 2. Account

账户表示钱或债务所在的位置。

字段：

- `id`
- `name`
- `type`
- `balance`
- `currency`
- `isActive`
- `createdAt`
- `updatedAt`

账户类型：

- `cash`
- `bank`
- `wechat`
- `alipay`
- `investment`
- `creditCard`
- `loan`
- `other`

- `accountId`

- `note`

## 6. JournalEntry

V0.1 用户界面可以不展示分录，但底层需要预留复式记账结构。

字段：

- `id`
- `transactionId`
- `date`
- `description`
- `lines`
- `createdAt`
- `updatedAt`

`lines` 字段包含：

- `accountCode`
- `accountName`
- `debit`
- `credit`
- `reportElement`

约束：

- 每条 `JournalEntry` 的借方合计必须等于贷方合计。
- 简易版输入不要求用户理解借贷。
- V0.1 可以先生成简化分录或只预留结构，但不得破坏未来扩展。

## 7. ReportPeriod

字段：

- `id`
- `type`
- `startDate`
- `endDate`
- `label`

周期类型：

- `day`
- `week`
- `month`
- `quarter`
- `year`
- `custom`

V0.1 默认使用 `month`。

## 10. Storage Key

V0.1 通过 storage adapter 使用：

- `imcfo.accounts`
- `imcfo.transactions`
- `imcfo.assets`
- `imcfo.liabilities`
- `imcfo.journalEntries`
- `imcfo.settings`

## 11. 税务字段预留

V0.1 可预留：

- `taxCategory`
- `taxWithheld`
- `taxPeriod`
- `beforeTaxAmount`
- `afterTaxAmount`
- `socialInsuranceAmount`
- `housingFundAmount`
- `deductionType`
- `isTaxRelated`

这些字段只用于记录和后续扩展，不用于正式申报。

## 12. V0.1 暂不建模

V0.1 暂不建模：

- 企业科目体系
- 个体工商户经营账户
- 发票
- 增值税
- 家庭成员
- 多主体合并
- 云端用户
