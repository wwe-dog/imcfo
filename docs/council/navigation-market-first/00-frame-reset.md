# Market-first Navigation Task Force - Frame Reset

> 本轮是市场视角导航战略讨论，不是现有导航检查，不是代码实现，不是 Constitution / specs 修改。

## 1. Task Force

本轮沿用 IMCFO Constitution v2 Council，不创建新的 Council。临时专题小组为：

Market-first Navigation Task Force / 市场视角导航战略小组。

主导 Agent:

- External Intelligence Agent
- User Behavior Agent
- Product Doctrine Agent
- AI Input Agent
- Originality Judge

辅助 Agent:

- Financial Core Agent
- Architecture Contract Agent
- Scope Judge
- Innovation Freedom Agent
- Spec Boundary Agent

弱化 Agent:

- Codebase Discovery Agent
- Implementation Reality Critic

## 2. Reset Rule

本轮不以当前 IMCFO App 导航为起点。

本轮不讨论当前 App 导航怎么改，不读取现有 navigation / screens 代码来决定方案，不把当前页面结构作为约束。

讨论出发点是：

- 用户为什么会下载一个 AI-first Personal CFO App。
- 用户打开 App 的第一动机是什么。
- 普通记账 App 为什么同质化。
- AI-first 输入如何改变产品入口。
- 生活事件如何被翻译成财务语言。
- 导航如何让用户感知 IMCFO 不是普通记账 App。

## 3. Current Implementation Boundary

当前实现只能在未来 implementation phase 中作为可落地性约束被重新评估。

本轮结论不得直接写入 constitution 或 specs，不得修改 mobile / backend / package / lockfile，不得自动提交。

## 4. Strategic Question

如果今天从 0 设计一个 AI-first Personal CFO App，IMCFO 的一级导航、全局主操作、信息架构、核心用户路径应该是什么？
