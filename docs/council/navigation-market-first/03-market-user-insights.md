# Market and User Insights

## 1. User Download Motivation

Users do not download a Personal CFO app because they want more bookkeeping work.

They download it because they want to reduce financial uncertainty:

- Am I safe this month?
- Can I spend this money?
- Did my financial state improve?
- Where did the money leak?
- What should I do next?
- Can I trust the numbers?

This means IMCFO's first screen should not feel like a ledger maintenance tool. It should feel like a financial operating brief.

## 2. Why Ordinary Bookkeeping Apps Become Homogeneous

Ordinary bookkeeping apps often organize around objects:

- transactions
- accounts
- categories
- budgets
- reports
- settings

This is easy to implement, but it pushes interpretation work back to the user. The user records facts, then must personally infer cash safety, profitability, debt pressure, spending efficiency and next action.

IMCFO should not compete by being a prettier ledger. It should compete by translating life events into financial consequences and action.

## 3. Why Users Abandon Bookkeeping

Evidence and market patterns point to five likely causes:

- Manual entry feels like homework.
- Feedback arrives too late or feels generic.
- Reports look professional but do not tell the user what to do.
- The user does not trust automatic categorization or AI interpretation.
- The product asks for ongoing discipline before giving obvious value.

Navigation must therefore minimize entry friction, give immediate post-confirmation impact, and keep a traceable path from insight back to posted facts.

## 4. AI-first Product Entry

AI-first does not mean "add an AI tab".

It means:

- The user can start from natural language.
- The system translates messy input into a candidate financial draft.
- The user confirms before anything becomes fact.
- After posting, the system explains impact using the rule layer and report engine.
- The same AI capability can answer questions across cockpit, reports and ledger, but it cannot bypass financial rules.

AI input should be a global primary action and system capability, not merely a destination.

## 5. Personal CFO Navigation Principle

IMCFO navigation should represent the user's financial operating loop:

1. Capture a life event.
2. Translate it into financial language.
3. Confirm facts.
4. Update trusted reports.
5. Surface operating state.
6. Recommend optional action.

This loop is more important than copying any competitor tab structure.

## 6. Market-first Strategic Tension

The strongest market-facing idea is "经营自己".

The strongest usability risk is that "经营" can become too abstract for ordinary users.

The navigation must therefore pair CFO language with concrete user questions:

- "经营" should answer what changed, what matters, and what to do.
- "报告" should preserve formal financial truth.
- "账本" should preserve traceability.
- "说一笔" should remain globally accessible as the fastest entry.
