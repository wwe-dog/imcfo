# Navigation Model Critiques

## 1. Product Contrarian Agent

Largest attacks:

- CFO Cockpit risks becoming a dashboard costume over a bookkeeping app.
- Operating System sounds big but may not mean anything to ordinary users.
- AI-first Input is differentiated, but if it becomes a chat shell it weakens financial credibility.
- Financial Statement Model is rigorous but can feel like homework.
- Life Event Translation is strongest conceptually, but "看影响" must not imply final accounting truth.
- Coach / Advisor can be marketable, but may create advice liability and generic nagging.
- Operating CFO Loop is the most balanced, but "经营" must be tested for comprehension.

Required revision:

- Do not use "记账" as a top-level identity.
- Use ordinary-language proof points: safe, changed, impact, action, truth.
- Keep the Personal CFO identity visible through operating loop, not through accounting jargon.

## 2. Friction & Retention Critic

Largest attacks:

- Any model that hides input will fail because manual entry is the highest-friction behavior.
- Any model that starts with reports will fail for new users with sparse data.
- A daily cockpit must give value even before the user has weeks of history.
- "经营" is sticky for product identity but may require first-run explanation.

Required revision:

- Make "说一笔" globally available.
- After posting, immediately show financial impact and next best destination.
- Empty states must invite useful setup, not show fake numbers.

## 3. Accounting Red-Team Agent

Largest attacks:

- AI-first Input Model can confuse raw input, draft, posted transaction and report fact.
- Life Event Translation can over-promise if "impact" is not labeled as estimate before posting.
- Coach / Advisor can mix suggestions with financial facts.
- Financial Statement Model is safest but least market-friendly.

Required revision:

- Navigation must preserve four states: raw input, candidate draft, posted fact, report result.
- Reports must remain authoritative after posting.
- Advice must trace back to posted facts or explicitly mark itself as hypothesis.

## 4. AI Failure-Mode Critic

Failure cases:

- "今天早餐 18，午餐 32，都用支付宝" needs split draft handling.
- "我还了花呗 500" must not become an expense twice.
- "买了基金 3000" may be investment transfer, not consumption.
- "朋友转我 200，我又请他吃饭 80" contains multiple directions and possibly reimbursement / social expense.

Navigation risks:

- If AI input is a tab, failures may trap the user in one page.
- If AI input is only hidden in a cockpit, users may miss it.
- If unconfirmed drafts appear in the ledger, users may believe they affected reports.

Required revision:

- Use a global input action with a visible "待确认" recovery state.
- Failed recognition returns to editable draft / text input, not to nowhere.
- Drafts never count as posted facts.

## 5. Overengineering Critic

Largest attacks:

- Operating System Model is too broad for MVP.
- Financial Statement Model can force full accounting infrastructure before market learning.
- Coach / Advisor Model can trigger a huge advice and responsibility system.
- Too many clever labels can reduce comprehension.

Required revision:

- Start with four destinations plus one global action.
- Treat "Operating System" as long-term language, not current IA.
- Do not split every financial object into top-level navigation.

## 6. Evidence Quality Critic

Largest attacks:

- Competitor evidence proves market patterns, not final IMCFO labels.
- App Store descriptions are marketing copy.
- US household finance stress does not directly represent Chinese students or early-career users.
- AI assistant product pages do not prove retention.

Required revision:

- Mark final recommendation as strategy to test, not permanent rule.
- Keep China user testing as next required validation.
- Do not claim "market proves" where evidence only suggests.

## 7. Model-by-model Verdict

| Model | Verdict | Main reason |
|---|---|---|
| A. CFO Cockpit | Revise | Strong state/action frame, but must avoid chart wall and business-heavy naming. |
| B. Operating System | Downgrade | Strategic vision only; too broad for current validation. |
| C. AI-first Input | Revise | Good input breakthrough, weak if it becomes the whole IA. |
| D. Financial Statement | Downgrade | Needed as trust layer, not primary consumer navigation. |
| E. Life Event Translation | Revise | Strong differentiator, but needs report authority boundaries. |
| F. Coach / Advisor | Revise | Useful explanation layer, risky as primary identity. |
| G. Operating CFO Loop | Accept with caution | Best balance of differentiation, trust and simplicity; needs label testing. |
