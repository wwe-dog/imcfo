# Evidence Quality Review

Evidence Quality Critic reviewed the market-first research under the rule: evidence can inspire IMCFO, but cannot override IMCFO's product soul, financial boundaries, or local-first privacy direction.

## 1. Overall Assessment

Result: usable with caution.

The evidence meets the minimum quantity requirement and covers more than 20 sources, more than 8 product cases, China / Asia references, international references, AI-first input, mobile UX, and CFO dashboard / business finance references.

The strongest evidence supports task priority, not a literal navigation answer. It proves that users need low-friction input, financial certainty, trustable records, cash-flow awareness, and action-oriented summaries. It does not prove that a specific IMCFO tab list is universally correct.

## 3. Rejected or Downgraded Uses

- Copying YNAB's budget-first navigation is rejected. It would narrow IMCFO into a budgeting methodology product.
- Copying Rocket Money's subscription-first focus is rejected. Recurring obligations are important but not the whole Personal CFO loop.
- Copying Cleo's personality-first chatbot is rejected. It risks reducing financial trust and auditability.
- Copying SMB accounting navigation is rejected. Personal users should not navigate invoices, bills, reconciliation, and forecasts like a company controller.
- Treating App Store copy as proof of user retention is rejected. It is only product positioning evidence.
- Treating community/search evidence as quantitative proof is rejected. It remains hypothesis.

## 4. Usable Strategic Signals

- Top-level navigation should represent recurring user decisions, not database entities.
- AI input should be available globally, but AI output must remain draft / estimate until confirmed.
- A trust layer is required: users need a place to inspect posted transactions, accounts, assets and liabilities.
- A formal report layer is required, but it should not dominate first-time user mental models.
- A cockpit / brief layer is valuable if it is decision-oriented, not a chart wall.
- Low-friction input is necessary but should not become the entire product identity.

## 5. Evidence Gaps

- China-specific primary research on why young users abandon bookkeeping apps is still missing.
- No source directly proves the best IMCFO navigation labels.
- AI-first finance navigation remains emerging; many claims are product marketing.
- User trust around AI financial estimates requires usability and safety testing.
