## 3. Scope Judge

Scope rulings:

- No login, payment, tax filing, enterprise subject, or cloud ledger is introduced by Constitution v2.
- Backend remains AI / ASR service boundary, not ledger database.
- AI auto-posting is rejected.
- Enterprise accounting workflows may inspire review gates but do not expand MVP scope.

## 4. Contradiction Judge

Contradiction rulings:

- No current tech stack is written as permanent rule.
- No current page structure is written as permanent rule.
- AI Draft and posted transaction are explicitly separated.
- UI formula creation is explicitly forbidden.
- Old rules are not restored.
- Discovery facts and Constitution rules are separated by file tree.

## 5. Originality Judge

Originality rulings:

- Generic “AI bookkeeping” is insufficient and rejected as core positioning.
- The distinctive product language is “personal CFO loop” and “life events translated into company-style financial language.”
- Visual identity may support originality, but only if it strengthens financial understanding.

## 6. Innovation Freedom Agent

Innovation rulings:

- Future tech stack, storage, AI vendor, page structure, visual language, and business model may change through defined governance.
- Current V0.1 implementation details are not frozen.
- Experiments remain allowed but cannot override core financial / AI boundaries.

## 7. Spec Boundary Agent

Boundary rulings:

- Current files, dependencies, screens, storage keys, endpoints: discovery/specs.
- Product soul, financial safety, AI write boundary, change governance: constitution.
- Current visual direction and collaboration roles: doctrine.
- Layer responsibilities and forbidden crossings: contracts.
- Future directions: experiments.
- Major decisions: ADR.
