# Candidate Directions

> This file is a candidate backlog: directions that may be promoted into experiments, specs, or ADRs in the future. It is not an experiment record, not a roadmap, and not a commitment. Items here have not been accepted into core scope unless explicitly moved to a spec or ADR.

## 1. AI Input

- Multi-transaction Draft splitting.
- “Need more info” Draft flow.
- AI explanation that cites report engine output.
- OCR receipt input.
- WeChat / Alipay bill import assistant.

## 2. Financial Core

- Journal entry generation from transaction rules.
- Import replay validation.
- Transfer and repayment hardening.
- Investment transaction modeling.
- Report consistency tests.

## 3. Product Feedback

- Weekly personal operating review.
- Monthly CFO letter.
- Cash pressure warning.
- Net worth and cash-flow trend interpretation.
- Action suggestions based on financial state.

## 4. Visual Experience

- Data-bound CFO sphere.
- Operating health HUD.
- Low-motion mode.
- Light mode variant.
- More restrained professional report mode.

## 5. Collaboration

- Task-specific Council mini-rounds.
- External intelligence refresh protocol.
- Context compression snapshot automation using `docs/context/current-project-context.md`.

## 6. Not Yet Accepted

These ideas are not accepted into core scope:

- backend ledger database
- automatic AI posting
- tax filing
- enterprise accounting
- payment services
- login and cloud sync by default
