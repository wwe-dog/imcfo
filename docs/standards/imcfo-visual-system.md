# IMCFO 视觉系统规范：暗黑液态 CFO 风格

最近同步：2026-05-07  
适用范围：IMCFO 移动端 V0.1 的首页、核心输入入口、HUD、报表、管理页、设置页和后续设计任务。  
规则级别：项目视觉规则。若本文件与旧的暖色/橙色视觉描述冲突，以本文件为准；若涉及会计、存储、交易映射、报表口径，以 `docs/00-project-constitution.md` 和会计规则文档为准。

## 1. 视觉总定位

IMCFO 不再被限制为传统“暖色个人金融 App”或“橙色记账 App”。

新的主视觉方向为：

> IMCFO 暗黑液态 CFO 风格

英文可称：

> Dark Liquid CFO Style

一句话定义：

> 深色金融操作台 + 液态玻璃卡片球体 + 赛博 HUD 数据流 + 个人 CFO AI 入口。

完整定义：

> 以深色数字金融背景为底座，通过液态玻璃卡片、HUD 数据流、空间化财务球体和高对比财务语义色，构建一个 AI 时代的个人 CFO 操作系统。

产品气质关键词：

- 深色
- 高级
- 未来感
- AI 感
- 玻璃态
- HUD
- 数字金融
- 仪表盘
- 球体入口
- 空间层级
- 个人 CFO 操作系统

## 2. 视觉结构规则

首页、智能记一笔、AI 输入入口和核心经营仪表盘可以使用三层结构。

### 2.1 深色金融背景

- 使用黑色、深石墨黑、夜色黑、深蓝黑作为系统底色。
- 允许低透明几何网格、金融网络线、空间背景纹理。
- 背景必须低干扰，不能影响文字、金额、按钮和财务状态的可读性。
- 背景服务于数字金融操作台的纵深，不是纯装饰。

### 2.2 顶部 HUD 数据带

- 可展示收入、本月净收益、支出、经营状态等核心指标。
- 可使用横向扫描线、微弱流光、数据节点、细线图形。
- HUD 是财务状态感知层，不是纯装饰层。
- HUD 信息必须可读，不能用动效或透明度削弱关键金额。

### 2.3 中心液态玻璃财务球体

- 由多张半透明功能节点卡片组成。
- 卡片代表账户、资产、负债、收入、支出、报表、记一笔等财务功能节点。
- 球体是 IMCFO 的核心视觉符号和 AI 输入入口。
- 球体可以拖动、旋转、展开，但不能牺牲可用性和性能。
- 球体交互必须保留明确点击目标、焦点状态和降级展示方案。

## 3. 颜色治理规则

IMCFO 不再限制只能使用暖色和橙色。橙色保留为品牌锚点、关键行动色和警示色之一，不再是唯一视觉主色。

新的颜色体系分为三层。

### 3.1 系统底色层

可使用：

- 深石墨黑
- 夜色黑
- 深蓝黑
- 暖白 / 米白
- 低透明网格线

用途：

- 首页背景
- AI 输入背景
- 数字金融操作台
- 视觉纵深

### 3.2 品牌与数字金融表现层

可使用：

- 橙金 / 琥珀金
- 青蓝 / 水绿色
- 薄荷绿
- 紫蓝 / 电光紫
- 灰白 / 银白

用途：

- 首页球体
- HUD 数据带
- 液态玻璃卡片
- AI 输入入口
- 当前 tab
- 关键行动按钮
- 视觉高光

### 3.3 财务语义层

这层必须稳定，不能随意变化：

- 正向 / 收入 / 改善 / 收益：绿色。
- 负向 / 支出 / 风险 / 亏损：红色。
- 警示 / 待处理 / 重点提醒：橙色或琥珀色。
- 中性金额 / 普通资产 / 普通文本：黑色、深灰或浅灰。
- 链接 / 下钻 / 可点击：蓝色或品牌强调色。

硬性约束：

- 同一个财务含义不能在不同页面使用互相矛盾的颜色。
- 禁止用红色表示收入。
- 禁止用绿色表示风险。
- 禁止为了视觉炫酷牺牲财务语义一致性。

## 4. 模块适用范围

### 4.1 首页 / 球体 / AI 输入 / HUD

允许：

- 暗黑背景
- 液态玻璃
- 发光边框
- 半透明卡片
- 空间层级
- 轻微流光
- 数据扫描线
- 球体旋转
- 未来感图标

目标：让用户感觉自己进入的是一个个人 CFO 操作系统，而不是普通记账本。

### 4.2 管理页 / 账户 / 资产负债 / 交易记录

必须克制：

- 清晰优先。
- 可读优先。
- 线分隔列表优先。
- 不要堆叠大量玻璃卡片。
- 不要使用过多发光和透明叠层。
- 不要影响交易、账户、资产、负债信息的识别。

管理页可以继承深色或高级金融气质，但不能牺牲财务底账的清晰度。

### 4.3 报表页

必须强调：

- 数字可读性。
- 财务语义色。
- 指标层级。
- 简易版 / 专业版表达一致。
- 不得为了视觉效果改动报表口径。

报表页可以高级，但不能过度装饰。

### 4.4 我的 / 设置

保持低干扰：

- 中性色为主。
- 操作清晰。
- 不做强视觉炫技。

## 5. 组件风格规则

新的核心组件语言为：

- Liquid Finance Chip
- Glass Finance Tile
- CFO Node Card

中文可称：

- 液态财务芯片
- 玻璃财务节点
- CFO 功能节点卡

组件特征：

- 半透明背景
- 高圆角
- 柔和边框
- 轻微内发光
- 轻微阴影
- 图标悬浮
- 层级叠加
- 中心聚合
- 深浅透明度表达空间距离

禁止：

- 随机彩虹色
- 高饱和色大面积堆叠
- 过重阴影
- 多层 blur 嵌套
- 大量透明层覆盖文字
- 游戏化 UI
- 与财务系统不匹配的娱乐化视觉

## 6. 动效规则

允许的动效方向：

- 悬浮
- 旋转
- 聚合
- 展开
- 吸附
- 扫描
- 呼吸光
- 数据流动

动效约束：

- 动效必须服务于理解和输入。
- 球体动效不能妨碍点击和阅读。
- HUD 动效必须轻量。
- Android 端必须控制 blur、阴影、透明叠层、图标数量，避免性能灾难。
- 不得声称达到 120 FPS，除非经过真实性能数据验证。

## 7. 与产品定位的关系

新的视觉方向必须服务于 IMCFO 的产品定位：

> IMCFO 不是普通记账 App。IMCFO 是把普通自然人的个人生活翻译成公司式财务报表的个人 CFO 系统。

核心表达仍然是：

> 像经营公司一样经营自己。

视觉应该传达：

- 我不是在记流水账。
- 我是在管理一家“个人公司”。
- 系统正在帮我理解现金流、利润、资产负债和经营状态。
- AI 输入入口是个人 CFO 系统的核心入口。

## 8. 保留的产品红线

即使视觉升级，也必须保留以下规则：

- 用户可见文案必须中文。
- 底部导航固定：首页 / 管理 / 报表 / 我的。
- UI 不得发明会计公式。
- Screen 不得直接调用 AsyncStorage。
- 报表计算函数必须保持纯函数。
- 不新增后端、登录、数据库、云同步、支付、会员、税务、VAT、发票、个体工商户或企业主体逻辑。
- 真实 AI/API 是否进入 V0.1 需要单独决策，不能因为视觉写成 AI 风格就自动接入真实 API。

## 9. 旧表述替换规则

需要替换或降级的旧表述：

- “暖色中国个人金融 + CFO 仪表盘”
- “橙色主色”
- “暖色金融视觉”
- “只能使用暖色 / 橙色体系”

新表述：

- “IMCFO 暗黑液态 CFO 风格”
- “深色数字金融操作台”
- “液态玻璃财务球体”
- “HUD 数据流”
- “橙色保留为品牌锚点和关键行动色，不再是唯一视觉主色”
- “财务语义色保持稳定，品牌表现色适度开放”

注意：不是完全删除暖色和橙色，而是把它们从“唯一限制”降级为“品牌锚点 / 行动强调 / 警示色”。
