# 我为 CFO · V0.1 页面结构

V0.1 只做 5 个页面。页面只负责输入、展示和导航，不负责发明会计公式。

## 1. 页面列表

1. 首页 Dashboard
2. 记一笔 Record
3. 资产负债 Assets & Liabilities
4. 报表 Reports
5. 设置 Settings

## 2. 首页 Dashboard

首页回答：

我现在这家“个人公司”的财务全貌是什么？

首页不是复杂诊断页，不是 AI 建议页，不是消费排行榜。

首页必须展示 6 个基础财务项目：

- 资产
- 负债
- 所有者权益
- 收入
- 费用
- 利润

首页可以展示辅助指标：

- 资产负债率
- 储蓄率
- 本期现金净变化
- 经营活动现金流净额
- 投资活动现金流净额
- 筹资活动现金流净额

首页可以保留轻量提示：

- 本月利润为正
- 本月利润为负
- 本月现金减少
- 本月负债上升

V0.1 首页暂不做：

- 经营评分
- AI 诊断
- 复杂建议
- 风险评级
- 自动生成月报

## 3. 记一笔 Record

记一笔不是普通流水输入页。

它的目标是把用户的个人经济事项转成可用于三大报表的数据。

V0.1 支持记录类型：

- 收入
- 费用
- 资产增加
- 资产减少
- 负债增加
- 负债减少
- 转账
- 投资买入
- 投资卖出
- 还款

输入顺序建议：

1. 先选类型。
2. 再填金额。
3. 再选账户。
4. 再选分类。
5. 最后写备注。

V0.1 不要求用户手动写借贷分录。

## 4. 资产负债 Assets & Liabilities

此页面回答：

我现在拥有多少，欠了多少？

V0.1 管理资产：

- 现金
- 银行卡余额
- 微信余额
- 支付宝余额
- 投资资产
- 应收款
- 固定资产
- 其他资产

V0.1 管理负债：

- 信用卡
- 花呗/白条
- 借款
- 贷款
- 应付款
- 其他负债

固定资产 V0.1 可以手动录入，但不强制折旧。

信用卡必须作为负债账户处理。

## 5. 报表 Reports

报表页是核心页面。

V0.1 必须支持：

- 资产负债表
- 利润表
- 现金流量表

每张报表都支持：

- 简易版
- 专业版

重要原则：

- 底层数据只有一套。
- 计算逻辑只有一套。
- 展示方式分成两套。

## 6. 设置 Settings

V0.1 设置页只做基础配置和数据管理：

- 默认货币，默认人民币
- 默认报表周期，默认本月
- 默认展示模式，简易版或专业版
- 数据导出
- 数据导入，后续可选
- 清空本地数据
- 示例数据开关，开发阶段可选

清空数据必须二次确认。

导出数据必须包含账户、交易、资产、负债、设置和版本号。

## 7. UI 文案规则

- 用户看到的文案必须是中文。
- 技术命名可以使用 English。
- 简易版使用生活语言。
- 专业版使用财务语言。
- UI 不能展示未经计算层确认的公式结果。
