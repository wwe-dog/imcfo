# 我为 CFO · V0.1 交易映射规则

本文档定义典型交易如何影响资产、负债、收入、费用、利润和现金流。实现交易逻辑时应优先查本文件。

## 1. 映射原则

- 每笔交易必须明确影响哪些会计要素。
- 每笔交易必须明确现金流类型。
- UI 只负责收集用户输入，不负责临时创造会计处理。
- 异常交易必须先补充到本文档，再进入实现。
- 简易版输入可以口语化，底层映射必须稳定。

## 2. 现金流类型

- `operating`：经营活动现金流
- `investing`：投资活动现金流
- `financing`：筹资活动现金流
- `nonCash`：不产生现金流的事项

## 3. 典型交易规则

### 3.1 工资到账

影响：

- 资产增加
- 收入增加
- 经营活动现金流入

现金流类型：`operating`

### 3.2 税后工资到账

影响：

- 资产增加
- 收入增加，按税后收入记录
- 可选记录税前收入、个税、社保、公积金字段
- 经营活动现金流入

现金流类型：`operating`

### 3.3 餐饮消费，现金支付

影响：

- 费用增加
- 资产减少
- 经营活动现金流出

现金流类型：`operating`

### 3.4 学习课程支出

影响：

- 费用增加
- 资产减少
- 经营活动现金流出
- 简易版标注为成长投入

现金流类型：`operating`

### 3.5 买电脑

用户可选择作为费用或固定资产。

作为费用：

- 费用增加
- 资产减少
- 经营活动现金流出
- 现金流类型：`operating`

作为固定资产：

- 固定资产增加
- 现金资产减少
- 投资活动现金流出
- 不直接影响利润表
- 现金流类型：`investing`

### 3.6 信用卡消费

影响：

- 费用增加
- 负债增加
- 不产生现金流出

现金流类型：`nonCash`

硬性规则：

信用卡消费不能被当作现金支出处理。

### 3.7 信用卡还款

影响：

- 负债减少
- 资产减少
- 筹资活动现金流出，或个人适配展示为债务偿还流出

现金流类型：`financing`

### 3.8 买基金

影响：

- 投资资产增加
- 现金资产减少
- 投资活动现金流出
- 不直接影响利润表

现金流类型：`investing`

### 3.9 卖基金盈利

影响：

- 现金资产增加
- 投资资产减少
- 投资收益增加，按盈利部分
- 投资活动现金流入

现金流类型：`investing`

### 3.10 借给别人钱

影响：

- 应收款增加
- 现金资产减少
- 通常不计入费用

V0.1 暂定：

- 现金流类型使用 `investing`。
- 后续如要改成其他现金流口径，必须先修改本文档。

### 3.11 别人还钱

影响：

- 现金资产增加
- 应收款减少
- 通常不计入收入

V0.1 暂定：

- 现金流类型使用 `investing`。
- 与“借给别人钱”的现金流口径保持一致。

### 3.12 借入一笔钱

影响：

- 现金资产增加
- 负债增加
- 筹资活动现金流入

现金流类型：`financing`

### 3.13 偿还借款本金

影响：

- 现金资产减少
- 负债减少
- 筹资活动现金流出
- 不影响利润表

现金流类型：`financing`

### 3.14 支付借款利息

影响：

- 费用增加，财务费用
- 现金资产减少
- 筹资活动现金流出，或个人适配展示为债务成本现金流出

现金流类型：`financing`

### 3.15 退款

V0.1 暂定：

- 优先作为原费用的冲减。
- 如果无法关联原费用，可作为“退款”收入分类展示。

实现时必须保持同一口径，不允许不同页面各自处理。

## 4. 后续版本规则

以下内容不进入 V0.1：

- 投资估值变动
- 固定资产自动折旧
- 坏账准备
- 信用卡分期复杂拆分
- 个体工商户经营成本
- 发票与增值税
