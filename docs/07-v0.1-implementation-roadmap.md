# 我为 CFO · V0.1 实施路线图

本文档定义从当前文档阶段到可运行 MVP 的开发顺序。

## 1. 当前阶段

当前只做文档和规则整理。

不得在本阶段做：

- 创建 React 应用
- 安装依赖
- 添加后端
- 添加登录、数据库、AI、支付、API 集成

## 2. 推荐开发顺序

1. 创建 Expo + React Native + TypeScript 项目。
2. 建立基础目录结构。
3. 定义 TypeScript 数据模型。
4. 实现 AsyncStorage storage adapter 读写层。
5. 实现交易映射规则。
6. 实现报表纯计算函数。
7. 添加最小公式测试。
8. 实现页面骨架。
9. 实现首页 Dashboard。
10. 实现记一笔页面。
11. 实现资产负债页面。
12. 实现报表页简易版。
13. 实现报表页专业版。
14. 实现设置页导出和清空。
15. 运行 build/type check。

## 4. 第一轮实现目标

第一轮只追求跑通：

- 有 Expo + React Native + TypeScript 项目
- 有基础页面导航
- 有 TypeScript 数据模型
- 有 AsyncStorage storage adapter 存取
- 有纯报表计算函数
- 有几条示例数据
- 首页能展示六要素
- 报表页能展示三大报表的基础数据

## 5. 第一轮不做

第一轮不做：

- 复杂视觉设计
- 完整表单体验
- 图表系统
- AI 分析
- 月报自动生成
- 云同步
- 税务计算
- 个体工商户模式

## 6. 验收标准

第一轮完成时必须：

- `npm run build` 可通过。
- 如果有 type check 命令，必须通过。
- 报表计算函数有最小样例验证。
- UI 没有直接写会计公式。
- AsyncStorage 逻辑没有混入报表计算函数。
