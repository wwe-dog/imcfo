# Arco Design Pro 2.0 对我为 CFO 的设计启发

## 可借鉴的视觉原则

### 1. 数据层级

Arco Design Pro 的价值不在于装饰，而在于清晰的数据层级。对“我为 CFO”来说，页面应优先区分：

- 当前最重要的财务结果
- 支撑这个结果的二级指标
- 可下钻查看的明细
- 辅助说明与状态信息

移动端应避免把所有数据一次性堆在页面上，而是使用首页概览、二级详情、三级明细的逐层下钻方式。

### 2. 紧凑表格 / 列表密度

Arco 的后台表格适合参考其信息密度和对齐方式，但不能照搬桌面表格。移动端应转换为紧凑列表：

- 第一行：名称 + 金额
- 第二行：日期、状态或关键标签
- 更多字段放入详情页

交易记录应使用账本式行，不使用过厚卡片。

### 3. 仪表盘卡片分组

首页可以参考 Arco 的 dashboard 组织方式，但要保持移动端单模块聚焦：

- 一屏只显示一个主要分析模块
- 指标放在小 pill 或紧凑卡片中
- 图表和表格分层展示
- 避免堆叠大量卡片形成滚动后台页

### 4. 图表卡片组合

Arco 的图表卡片适合参考标题区、指标区、图表区和图例区的组合。用于“我为 CFO”时应保持：

- 图表紧凑
- 标题明确
- 单图只回答一个问题
- 图例和标注尽量靠近数据

### 5. 管理页组织

管理页可以借鉴后台系统的“管理中心”模式，但移动端不能变成复杂菜单。应采用：

- 高频入口放主页面
- 低频维护功能放“更多”
- 复杂管理对象使用二级 / 三级页面

### 6. 详情页信息层级

详情页应参考 Arco 的分区方式：

- 顶部：对象名称和关键金额
- 中部：核心结构或趋势
- 底部：字段明细

不要把编辑、删除、冲销等高风险操作混入普通查看区域。

## 如何适配到我为 CFO

### 移动优先

所有设计必须先服务手机屏幕。桌面后台中的侧边栏、顶部导航、多列大表格都不应直接复制。

### 保留当前视觉方向

继续使用当前 cream / green / purple 风格。Arco 只作为结构和密度参考，不改变产品视觉基调。

### 避免桌面侧边栏

“我为 CFO”底部导航保持：首页、管理、报表、我的。不得为了套用后台模板新增桌面侧边栏或新底部 Tab。

### 避免大型后台表格

移动端列表应压缩为账本行、账户行、资产负债行。详细字段进入详情页。

### 交易记录

交易记录应使用紧凑 ledger rows：

- 行内只保留标题、金额、日期和现金状态
- 类型、分类、账户、备注放入详情页

### 首页和报表

首页和报表可以借鉴 Arco 的图表卡片，但每个卡片必须服务明确财务问题：

- 资产负债结构
- 收支现金流
- 净资产趋势
- 三大报表

### 管理页面

账户、资产、负债、交易记录适合使用 Arco 式管理页结构，但必须移动端化为：

- 大类总览
- 分类详情
- 具体对象详情
