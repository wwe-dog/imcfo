# Reference Pages

## Dashboard / 工作台

为什么有用：
工作台页面通常展示多指标总览、图表卡片和关键任务入口，适合参考信息优先级和卡片组合方式。

可借鉴到：
首页。用于优化“收支现金流”和“资产负债结构”的模块层级，但不能照搬桌面多列布局。

## Data Visualization / 数据可视化

为什么有用：
数据可视化页面通常包含图表标题、核心指标、图例、注释和辅助筛选，适合参考图表卡片构成。

可借鉴到：
首页、报表、资产负债详情。用于优化 donut chart、line chart、趋势区和财报图表卡片。

## Table / 表格

为什么有用：
Arco 表格强调列对齐、状态标签、金额展示和密集信息排版。

可借鉴到：
交易记录、资产负债管理。移动端不使用大表格，应转换为紧凑列表行和详情页。

## List / 列表

为什么有用：
列表适合移动端复用，特别是交易记录、账户记录、资产负债条目。

可借鉴到：
交易记录、账户管理、资产负债管理。重点借鉴标题/副标题/状态/金额的排列方式。

## Detail Page / 详情页

为什么有用：
详情页有清晰的信息分区，适合展示对象摘要、字段明细和关联数据。

可借鉴到：
账户详情、资产明细、负债明细、交易详情、净资产详情。

## Form / 表单

为什么有用：
表单页面可以参考字段分组、必填校验、提示文案和操作按钮位置。

可借鉴到：
记一笔高级填写、账户新增、资产新增、负债新增、对账 / 资产盘点。

## Filter / 筛选

为什么有用：
筛选模式适合交易记录、报表和资产负债列表，但移动端需要简化。

可借鉴到：
交易记录。当前只做搜索和筛选入口，后续可扩展为底部弹层筛选。

## Management Center / 管理中心

为什么有用：
管理中心适合组织低频维护功能。

可借鉴到：
管理页“更多”、我的 / 数据管理。用于组织账户管理、资产负债管理、交易记录和对账入口。
